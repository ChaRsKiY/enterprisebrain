# Internationalization (i18n) Design — Enterprisebrain.eu

**Date:** 2026-04-16  
**Languages:** German (de, default) + English (en)  
**Stack:** Astro 6 built-in i18n, static output, Vercel deployment

---

## 1. Architecture

Astro's built-in i18n routing handles locale-based URL generation. All translatable strings live in typed TypeScript dictionaries. Components receive a `lang` prop and call a `t()` helper to look up strings.

No third-party i18n library is added — Astro's built-in routing + simple TS objects are sufficient for a marketing site of this size.

**New files:**
```
src/i18n/
  de.ts       ← German translation dictionary
  en.ts       ← English translation dictionary
  utils.ts    ← useTranslations(lang) helper
```

---

## 2. URL Structure

| Page          | German (default) | English       |
|---------------|-----------------|---------------|
| Home          | `/`             | `/en/`        |
| Use Cases     | `/use-cases/`   | `/en/use-cases/` |
| Datenschutz   | `/datenschutz/` | (DE only)     |
| Impressum     | `/impressum/`   | (DE only)     |
| 404           | `/404`          | (shared)      |

German is the default locale — no `/de/` prefix. English gets the `/en/` prefix.

**astro.config.mjs i18n config:**
```js
i18n: {
  defaultLocale: 'de',
  locales: ['de', 'en'],
  routing: {
    prefixDefaultLocale: false,
  },
}
```

## 3. Page Files

Two new pages are added for English. Existing German pages are unchanged in location.

```
src/pages/
  index.astro           ← unchanged (German /)
  use-cases.astro       ← unchanged (German /use-cases/)
  datenschutz.astro     ← unchanged (DE only)
  impressum.astro       ← unchanged (DE only)
  en/
    index.astro         ← new (English /en/)
    use-cases.astro     ← new (English /en/use-cases/)
```

Each English page is a thin wrapper: it imports the same components as the German page but passes `lang="en"`.

`src/i18n/utils.ts`:
```ts
import { de } from './de';
import { en } from './en';

const translations = { de, en };

export type Lang = 'de' | 'en';
export type Translations = typeof de;

export function useTranslations(lang: Lang) {
  return translations[lang];
}
```

## 5. Component Changes

Every component that contains text gets a `lang: Lang` prop:

```astro
---
import { useTranslations } from '../i18n/utils';
import type { Lang } from '../i18n/utils';

interface Props { lang: Lang; }
const { lang } = Astro.props;
const t = useTranslations(lang);
---

<h1>{t.hero.title} <span class="gradient-text">{t.hero.titleHighlight}</span></h1>
```

Components affected:
- Navbar, Footer, HeroSection, VideoSection, HowItWorksSection, ProductSection,
  UseCasesPreview, TestimonialsSection, SecuritySection, PartnersSection,
  MotivationSection, FAQSection, ContactSection, UseCaseCard

Components NOT affected (no user-visible text):
- BaseLayout (receives translated title/description as props from pages)

## 6. Language Switcher

Added to Navbar — a minimal `DE / EN` toggle button pair placed between the nav links and the CTA button.

```astro
---
import { getRelativeLocaleUrl } from 'astro:i18n';
---
<div class="flex items-center gap-1 text-xs font-display font-600">
  <a href={getRelativeLocaleUrl('de', currentPagePath)}
     class={lang === 'de' ? 'text-primary' : 'text-ash hover:text-carbon'}>DE</a>
  <span class="text-ash/30">/</span>
  <a href={getRelativeLocaleUrl('en', currentPagePath)}
     class={lang === 'en' ? 'text-primary' : 'text-ash hover:text-carbon'}>EN</a>
</div>
```

The switcher appears in both desktop nav and mobile menu.

`currentPagePath` is derived from `Astro.url.pathname` with the locale prefix stripped, so `/en/use-cases/` correctly maps to `/use-cases/` when switching to DE.

## 7. SEO & hreflang

`BaseLayout.astro` adds `hreflang` alternate links for pages that have both language versions:

```html
<link rel="alternate" hreflang="de" href="https://enterprisebrain.eu/" />
<link rel="alternate" hreflang="en" href="https://enterprisebrain.eu/en/" />
<link rel="alternate" hreflang="x-default" href="https://enterprisebrain.eu/" />
```

BaseLayout receives a `showAlternates: boolean` prop (default `true`). Legal pages (datenschutz, impressum) pass `showAlternates={false}`.

---

## 8. Vercel Deployment

No Vercel adapter required — the site stays `output: 'static'` (default). Vercel auto-detects Astro and runs `astro build`. The `dist/` folder is served as a static site.

`vercel.json` is not required. Astro's sitemap integration will automatically include both `/` and `/en/` routes.

---

## 9. Out of Scope

- CMS / admin panel for editing translations (separate task)
- Datenschutz / Impressum English versions (DE legal documents)
- Right-to-left language support
- Dynamic locale detection based on browser language
